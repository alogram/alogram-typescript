# Contributing to Alogram SDKs

Thank you for your interest in improving Alogram's SDKs!

## Hub & Spoke Model (Private Hub)

This repository is a **distribution-only mirror**. It is part of a "Hub & Spoke" architecture where the source of truth is maintained in our private internal monorepo.

Because this code is generated, we **cannot accept Pull Requests** directly in this repository. Any changes made here would be overwritten during our next automated release cycle.

## How to Contribute

We value your feedback and contributions! Please use the following workflow:

### 1. Report Bugs or Request Features
The best way to contribute is to **[Open an Issue](https://github.com/alogram/alogram-python/issues)** in this repository.
- Provide a clear description of the issue or feature.
- Include code snippets or error logs if applicable.

### 2. Our Process
- An Alogram engineer will review your issue.
- If accepted, we will implement the fix or feature in our private internal monorepo (either in the OpenAPI spec or the SDK templates).
- Once the internal change is merged, a new version will be automatically built and synchronized to this public repository.
